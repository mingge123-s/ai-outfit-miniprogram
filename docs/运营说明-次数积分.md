# 晚风问小程序 · 次数/积分系统 运营说明

> 当前为「不接真钱」的次数/积分阶段：靠免费额度 + 兑换码 + 管理员充值运营，
> 主体升级为个体/企业、开通微信支付后再接正式付费。本说明覆盖日常运营常用操作。

---

## 一、系统怎么算次数

每个用户每天能生成多少张穿搭图，由两部分组成：

1. **当日免费额度**：每人每天 `DAILY_FREE_LIMIT` 次（免费，不扣积分，每天 0 点重置）。
2. **积分**：当日免费用完后，每生成 1 张扣 `CREDITS_PER_GENERATION`（默认 1）积分。
   - 新用户注册自动送 `FREE_SIGNUP_CREDITS`（默认 3）积分。
   - 积分只能通过「兑换码」或「管理员充值」获得，永久有效、不清零。

生成扣费是「先预扣、成功才真扣、失败自动退回」：
- 生成失败 / 服务器重启中断的任务 **不会扣次数**（自动返还）。
- 并发/重复提交不会透支（后端强制校验，前端改数据没用）。

前端显示：
- 首页底部：`今日免费剩余 X/N 次 · 积分 Y（约可生成 Z 次）`
- 「我的」页：剩余积分、今日剩余次数、🎁兑换码充值入口

## 三、常用运营操作（在生产服务器执行）

生产服务器：`43.142.31.198`，服务端目录：
`/www/wwwroot/ai-outfit-miniprogram/server`

SSH 登录后进入目录：
```bash
cd /www/wwwroot/ai-outfit-miniprogram/server
```

### 1. 给指定用户加次数
```bash
node scripts/credits.js grant <用户ID> <次数> "备注"
# 例：给 3 号用户加 20 次
node scripts/credits.js grant 3 20 "客服补偿"
```
> 用户ID：让用户打开小程序「我的」页，最上面「账号 #数字」里的数字就是。

### 2. 查询某用户余额
```bash
node scripts/credits.js balance <用户ID>
```

### 3. 批量生成兑换码（发给用户自行兑换）
```bash
node scripts/credits.js gen-code <每个码的次数> <生成数量> [每个码可用次数=1]
# 例：生成 10 个「+20 次」的一次性兑换码
node scripts/credits.js gen-code 20 10
```
> 生成后会打印一串以 WF 开头的码，复制发给用户，在「我的→兑换码充值」输入即可。
> `每个码可用次数` 设成 100 就是「一码可被 100 个不同用户各用一次」，适合做活动通用码。

### 4. 查看已有兑换码使用情况
```bash
node scripts/credits.js list-codes
```

### 5. 调节各项参数（改 .env 后需重启）
```bash
# 编辑 .env，可改：
#   DAILY_FREE_LIMIT=10        每人每天免费次数
#   FREE_SIGNUP_CREDITS=3      新用户注册赠送积分
#   CREDITS_PER_GENERATION=1   每次生成消耗积分
pm2 restart ai-outfit
curl -s localhost:3012/health   # 确认正常
```
> 说明：调小 `DAILY_FREE_LIMIT`（比如改成 0 或 1）会让用户更快用到积分，
> 更利于变现；调大更利于拉新。改完立即生效，不影响老用户已有积分。
