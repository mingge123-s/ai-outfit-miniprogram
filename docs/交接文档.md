# AI 穿搭小程序 — 项目交接文档

本文档面向接手本项目的开发、运营与运维人员，说明项目现状、部署方式、日常操作和待办事项。更细的次数/积分操作见 [《次数与积分运营说明》](运营说明.md)。

## 1. 项目概述

**产品名称**：AI 穿搭（微信小程序）

**核心能力**：用户上传衣物单品（或从衣柜选取），可选上传模特全身照，由后端调用生图模型生成完整穿搭效果图；支持衣柜管理、套装收藏、今日天气搭配推荐、会员分级与积分体系。

**代码仓库**：`https://github.com/mingge123-s/ai-outfit-miniprogram`

**生产环境**：

| 项目 | 值 |
| --- | --- |
| 小程序 AppID | `wx4da1d4863ab8ae9a` |
| 后端 API | `https://api.mingge.asia/outfit` |
| 生产服务器 | `111.228.14.193`（腾讯云） |
| 进程管理 | PM2，进程名 `ai-outfit` |
| 数据目录 | `server/data/`（SQLite + 用户图片） |

## 2. 技术栈

| 层级 | 技术 |
| --- | --- |
| 前端 | 微信原生小程序（WXML / WXSS / JS） |
| 后端 | Node.js + Express（ES Module） |
| 数据库 | SQLite（`better-sqlite3`） |
| 生图 | 火山方舟豆包 Seedream 5.0 Pro（生产推荐） |
| 衣物分类 | 火山方舟豆包 Seed 2.0 Mini（低成本视觉模型） |

| 天气 | Open-Meteo（开发/非商用；商业上线需确认授权） |
| 小程序上传 | `miniprogram-ci`（在固定服务器上执行，避免 IP 白名单频繁变更） |

## 4. 功能清单

### 4.1 已实现功能

| 模块 | 功能 |
| --- | --- |
| 登录 | 微信 `wx.login` → 后端 `code2session` 换 token |
| 穿搭生成 | 异步任务队列；支持多单品参考图 + 可选模特照 + 背景风格 |
| 衣柜 | 批量上传（最多 9 张/次）、后台异步抠图 + 智能分类、按分类筛选、「全部」标签、多选后添加到生成 |
| 模特照 | 上传/管理个人形象照，生成时选用 |
| 套装收藏 | 收藏生成结果，支持重命名 |
| 生成历史 | 查看与删除历史记录 |
| 积分体系 | 每日免费次数 + 积分余额；预扣/失败返还 |
| 兑换码 | 用户自助兑换；管理员 CLI/HTTP 批量生成 |
| 会员分级 | 免费版 vs 会员，影响每日生图次数、衣柜/模特照/收藏上限 |
| 激励广告 | 看激励视频领取额外生图机会（需配置广告位 ID） |
| 今日搭配 | 模糊定位 → 实时天气 → 按场景从衣柜推荐完整穿搭 → 一键生成 |
| 管理接口 | 积分调整、会员设置、兑换码生成（需 `ADMIN_TOKEN`） |

### 4.2 权限额度（默认值，可在 `.env` 调整）

| 权限项 | 免费版 | 会员 |
| --- | ---: | ---: |
| 每日免费生图 | 3 次 | 10 次 |
| 衣柜单品上限 | 10 件 | 30 件 |
| 模特照上限 | 10 张 | 30 张 |
| 收藏套餐上限 | 20 套 | 40 套 |

超出每日免费次数后，每次生成消耗 `CREDITS_PER_GENERATION`（默认 1）积分。新用户注册赠送 `FREE_SIGNUP_CREDITS`（最新分支默认为 0）。

### 4.3 已搁置/隐藏功能

- **淘宝商品导入**：后端 `taobao.js` 仍保留，但前端入口已隐藏（合规与成本考量）。
- **微信支付**：套餐仅展示，`purchaseEnabled: false`，未接入真实支付（个人主体小程序限制）。

## 5. 部署与运维

### 5.1 服务器要求

- Linux（当前：腾讯云 Ubuntu）
- Node.js 18+
- Python 3 + `rembg`（衣柜自动抠图）
- PM2
- Nginx 反向代理（`/outfit` → `localhost:3000`）
- 磁盘：注意 `server/data/uploads/` 会持续增长，需定期清理或扩容

### 5.2 后端部署流程

```bash
# SSH 登录生产服务器后
cd /path/to/ai-outfit-miniprogram
git pull origin main          # 或合并后的目标分支
cd server
npm ci --omit=dev
pm2 restart ai-outfit --update-env
curl https://api.mingge.asia/outfit/health
pm2 logs ai-outfit --lines 100
```

健康检查应返回：

```json
{ "ok": true, "provider": "ark", "model": "doubao-seedream-5-0-pro-260628" }
```

### 5.3 环境变量（`server/.env`）

生产环境必须配置的关键变量：

```dotenv

MODEL_ID=doubao-seedream-5-0-pro-260628
ARK_SIZE=1152x2048

# 衣物自动分类（复用 ARK_API_KEY）

AUTO_CUTOUT=1

# 微信登录
WX_APPID=wx4da1d4863ab8ae9a
WX_SECRET=<小程序 AppSecret>

# 权限额度
FREE_DAILY_LIMIT=3
MEMBER_DAILY_LIMIT=10
FREE_WARDROBE_LIMIT=10
MEMBER_WARDROBE_LIMIT=30
FREE_PERSON_PHOTO_LIMIT=10
MEMBER_PERSON_PHOTO_LIMIT=30
FREE_OUTFIT_LIMIT=20
MEMBER_OUTFIT_LIMIT=40
FREE_SIGNUP_CREDITS=0

# 激励广告（配置广告位后开启）
REWARDED_AD_ENABLED=0
AD_REWARD_CREDITS=1

# 今日搭配（后端分析衣柜图片时需要公网可访问的 URL 前缀）
```

完整模板见 `server/.env.example`。

### 5.4 小程序发布流程

1. **配置合法域名**（微信公众平台 → 开发管理 → 开发设置）：
   - `request` 合法域名：`https://api.mingge.asia`
   - `downloadFile` 合法域名：`https://api.mingge.asia`

2. **配置上传白名单 IP**：在固定服务器 `111.228.14.193` 上执行上传，将该 IP 加入微信后台「代码上传 IP 白名单」。

3. **准备上传私钥**：从微信后台下载代码上传私钥，保存在服务器受控目录，权限 `chmod 600`，**绝不提交到 Git**。

4. **修改配置**：
   - `miniprogram/config.js` → `API_BASE_URL` 指向生产 API
   - 激励广告：填入 `REWARDED_VIDEO_AD_UNIT_ID`

5. **上传体验版**：

```bash
# 在已配置 miniprogram-ci 的服务器上
npx miniprogram-ci upload \
  --pp ./miniprogram \
  --pkp /path/to/private.key \
  --appid wx4da1d4863ab8ae9a \
  --uv <版本号> \
  --ud "<版本描述>"
```

6. 微信后台设为体验版 → 真机回归 → 提交审核 → 发布。

### 5.5 数据备份

核心数据：

- `server/data/app.db` — 用户、积分、衣柜、历史
- `server/data/uploads/` — 用户上传与生成图片

**备份方式**（二选一）：

```bash
# 方式 A：SQLite 在线备份（不停服）
sqlite3 server/data/app.db ".backup 'backup/app-$(date +%F).db'"

# 方式 B：停服后整目录拷贝
pm2 stop ai-outfit
tar czf backup/data-$(date +%F).tar.gz server/data/
pm2 start ai-outfit
```

建议每日自动备份并保留至少 7 天；数据库与图片目录必须同时备份。

## 6. 日常运营操作

### 6.1 查找用户

让用户打开小程序「我的」页，顶部显示 `账号 #123`，其中 `123` 即 `userId`。

### 6.2 常用 CLI 命令

在服务器仓库根目录执行：

```bash
# 查询积分余额
node server/scripts/credits.js balance 123

# 充值/扣减积分
node server/scripts/credits.js grant 123 20 客服补偿

# 生成兑换码（10 个，每个 +20 积分，限用 1 次）
node server/scripts/credits.js gen-code 20 10

# 查看最近兑换码
node server/scripts/credits.js list-codes

# 设置会员（30 天 / 永久 / 恢复免费）
node server/scripts/credits.js membership 123 member 30
node server/scripts/credits.js membership 123 member
node server/scripts/credits.js membership 123 free
```

更多细节见 [运营说明.md](运营说明.md)。

### 6.3 管理员 HTTP 接口

请求头需携带 `x-admin-token: <ADMIN_TOKEN>`：

- `POST /api/admin/credits/grant` — 调整积分
- `POST /api/admin/membership` — 设置会员
- `POST /api/admin/redeem-codes` — 批量生成兑换码

## 9. 关键实现说明

### 9.1 异步处理队列

- **生图任务**：`POST /api/tryon` 立即返回 `taskId`，后台队列执行；前端轮询 `GET /api/tryon/:id`。
- **衣柜处理**：上传后立即返回，后台 `wardrobeProcessQueue` 串行执行抠图 + 分类，避免并发压垮服务器。

### 9.2 积分预扣机制

提交生成时预扣积分；成功则确认消费，失败或服务重启中断则自动返还。失败的免费任务不占用当日免费次数。

### 9.3 生图构图

生产环境 `ARK_SIZE=1152x2048`（9:16 竖图），prompt 中强制要求完整全身、头部、面部、鞋子可见，避免裁切脸部。

### 9.4 今日搭配

1. 前端调用 `wx.getFuzzyLocation` 获取模糊坐标（不保存精确位置）。
2. 后端通过 Open-Meteo 获取实时天气。
3. 结合场景（日常/通勤/约会/运动/旅行）和用户衣柜，调用豆包 Mini 推荐完整穿搭。
4. 按日期 + 场景 + 粗粒度位置缓存，支持「换一套」。

### 9.5 前端文案规范

所有用户可见页面**不得出现 AI 模型供应商或模型名称**，统一使用「智能识别」「智能推荐」等中性表述。

## 11. 故障排查

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 登录失败 `40125 invalid appsecret` | `WX_SECRET` 错误或过期 | 微信后台重置 AppSecret 并更新 `.env` |
| 上传代码 `invalid ip` | 上传机 IP 不在白名单 | 将当前上传服务器 IP 加入白名单，或固定在 `111.228.14.193` 上传 |
| 衣柜上传后一直「处理中」 | `rembg` 未安装或队列阻塞 | 检查 `pm2 logs`；确认 Python/rembg 可用 |
| 分类失败 `ModelNotOpen` | 豆包视觉模型未开通 | 火山方舟控制台开通 `doubao-seed-2-0-mini-260215` |
| 生成图脸部被裁切 | 分辨率或 prompt 问题 | 确认 `ARK_SIZE=1152x2048` 且已部署最新 prompt 修复 |
| 今日搭配定位失败 | 模糊位置权限未开通 | 见「待办」；可用手动天气预设兜底 |
| 服务器内存/磁盘高 | 日志堆积、上传图片过多 | 清理日志与过期图片；已禁用异常的 `codepilot.service` |
| 生成超时 | 模型 API 慢或网络问题 | 查看 `pm2 logs`；检查火山方舟配额与账单 |

## 12. 安全清单

**不得提交到 Git 或发送到公开渠道的内容**：

- `server/.env`
- `WX_SECRET`、`ARK_API_KEY`、`ADMIN_TOKEN`
- 微信代码上传私钥（`.key` 文件）
- 生产数据库 `app.db` 与用户图片
- SSH 密码

如密钥曾泄露，立即在对应平台轮换。

服务器已配置 Fail2ban 防护 SSH 暴力破解。建议定期更新系统补丁。
